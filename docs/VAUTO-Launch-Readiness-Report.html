A) Kas veikia tobulai — paruošta testuotojams

AI ir klaidų tolerancija

  • Trumpų prekės ženklų guard'ai (vw, bmw, nike) — neblokuoja
  • Gemini interpretuoja rašybos klaidas, slangą, be diakritikų
  • callGeminiWithRetry su exponential backoff (429/503)
  • Zero-result apsauga, audience adaptation, B2B partner playbook

Spinta (Wardrobe)

  • Neaiškios nuotraukos → 200 + draugiškas pranešimas (ne 500 crash)
  • Kategorijų grupė be „Moterims" bias
  • Profilio importas su graceful fallback

B2B verslo kabinetas

  • Proaktyvus verslo partnerio vedimas (ne „Spinta tuščia")
  • Monetizacija: B2B boost 29.99€, lead 14.99€, Pro upsell
  • Insights, leads, Smart Boost — 22/22 deterministiniai testai

Pranešimai ir P2P pokalbiai

  • Wishlist → deep-link /listing/<slug>/ (ne home)
  • P2P abiem kryptimis → DB chat_message + Web Push + FCM fan-out
  • Notification Bell — polling fallback 8s (push denied) / 20s (granted)
  • Live chat — 4s server polling + BroadcastChannel + toast + garsas
  • SW deep-link → VAUTO_NAVIGATE → tikslus pokalbis + input autofocus
  • Pristatymas neblokuoja serverio (setImmediate, Promise.allSettled)

Infrastruktūra

  • Ops-secret middleware, carrier adapter contract, webhook idempotency
  • DB connected, embeddings synced, Stripe live

B) Įžvalgos / ko dar trūksta

Gemini 429 burst metu: Per testų seriją 4/10 AI probe'ų gavo HTTP 429. Retry veikia, bet per didelė apkrova vis tiek grąžina limitą. Beta metu stebėti rate limit.
FCM nekonfigūruotas: Android native push neveiks. Web Push veikia. Prieš APK beta — FIREBASE_SERVICE_ACCOUNT_JSON.
P2P latency ~4s: Nėra WebSocket — cross-device sync per polling. Messenger „akimirksniu" tik same-browser tab'uose. v1.1 — WebSocket/SSE.
Frontend tsc: 2 optional dep klaidos. Pridėti peer deps arba ts-expect-error.
Carrier API: Omniva/DPD/LP simulated — laukiami raktai (planuota).
Google OAuth / SMS: Neįjungta prod. Email/JWT auth veikia.
